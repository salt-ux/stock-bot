## 프로젝트
- 목표: 주식 자동매매 온보딩 실습 프로젝트
- 백엔드: Go
- 데이터베이스: MySQL

## 작업 원칙
- 기능은 단계(Phase)별로 점진적으로 구현한다. 한 번에 전체를 만들지 않는다.
- 각 단계는 `코드 + 테스트 + 간단 실행 가이드`까지 완료한다.
- 실제 증권사/데이터 제공자 교체가 가능하도록 어댑터 패턴을 유지한다.

## 권장 기술 스택
- HTTP API: Gin
- DB 접근: sqlc + MySQL
- 큐/비동기 작업: Redis + Asynq
- 스케줄러: robfig/cron
- 로깅: zap
- 메트릭: Prometheus
- 로컬 실행 환경: docker compose

## 브로커 연동 방침
- 실거래/시세 연동은 키움증권 REST API를 사용한다.
- Go 공식 SDK 지원 여부와 무관하게 HTTP/JSON 기반으로 Go 클라이언트를 직접 구현한다.
- 키움 연동 코드는 어댑터 인터페이스 뒤로 숨기고, 추후 타 증권사로 교체 가능하게 유지한다.

## 개발 순서(단계별)
1. 부트스트랩
- Go 모듈 초기화, 기본 프로젝트 구조, 설정 로더, 헬스체크 API 구성
- MySQL/Redis용 docker compose와 기본 Makefile 추가

2. 데이터 모델 + 영속 계층
- 스키마 정의: users, portfolios, positions, orders, fills, strategy_runs
- sqlc로 쿼리 코드 생성 후 리포지토리 계층 구현

3. 시세 데이터 어댑터
- 시세 데이터 인터페이스와 mock 제공자 1종 구현
- 캔들/호가 조회 및 캐시 정책 구현

4. 전략 엔진(모의투자)
- 전략 인터페이스와 1차 전략(예: 이동평균 교차) 추가
- 과거 캔들 기반 매수/매도 시그널 생성

5. 주문 시뮬레이션 + 리스크 통제
- 페이퍼 브로커 실행기 구현(실거래 없음)
- 보호 규칙 추가: 최대 포지션 크기, 일 손실 한도, 중복 주문 방지

6. 스케줄러 + 작업 큐
- 장 시간 이벤트와 주기 실행용 cron 작업 등록
- Asynq 워커를 통한 비동기 매매 파이프라인 실행

7. API + 운영 엔드포인트
- 포트폴리오, 포지션, 주문, 실행 이력, 전략 on/off API 제공
- 요청 검증 및 일관된 에러 응답 포맷 적용

8. 관측성 + 안정성
- 구조화 로그, 메트릭, 멱등 키, 재시도 정책, 타임아웃 적용
- DB/Redis 시작 점검 및 graceful shutdown 구현

9. 테스트 + 품질 검증
- 전략/리스크/주문 로직 단위 테스트 작성
- testcontainers 기반 통합 테스트(MySQL/Redis) 작성

10. 백테스트 + 리포트
- 과거 데이터 기반 간단 백테스트 실행기 구현
- 손익(PnL), 승률, 최대낙폭(max drawdown) 요약 리포트 생성
